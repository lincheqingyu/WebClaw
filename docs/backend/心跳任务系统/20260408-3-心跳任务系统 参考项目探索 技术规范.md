# 心跳任务系统参考项目探索

更新日期：2026-04-08

## 1. 探索目标

这份文档用于约束后续对 OpenClaw 和 Claude Code 心跳任务设计的探索方向。

## 2. 必答问题

每次探索至少回答下面 5 个问题：

1. 心跳任务的触发条件是什么
2. 任务状态如何表示
3. 任务如何避免重复执行
4. 任务如何恢复或中断
5. 主动任务如何使用已有上下文和记忆

## 3. 优先参考对象

- OpenClaw 中的主动任务 / 调度设计
- Claude Code 中与心跳、主动任务、后台推进相关的实现

## 4. 输出要求

每次探索应至少产出：

- 状态机草图
- 触发器分类
- 最小可复刻点
- 当前不建议做的点

## 5. 当前探索边界

这份探索文档只为后续心跳任务路线提供输入，不直接替代正式开发规划。
