# 设置面板浮层重构（最终版）

> 本文档为交付给 Claude Opus 的改造指令。要求**从根本上切换实现模式**，不打补丁、不在旧架构上堆叠修复。
>
> **版本说明**：本文档为 v2 草案的评审最终版，与初稿相比有以下关键调整：
> - **从模态改为非模态浮层**：移除全屏透明遮罩与"点击外部关闭"行为
> - **新增对话区空白挤占**：对话区内容容器 `padding-right` 与浮层 transform 同步动画
> - **新增 a11y 必备项**：Escape 键关闭、`inert` 隔离关闭态焦点
> - **同时改动两个文件**：`SettingsDrawer.tsx`（浮层化）+ `HomePageLayout.tsx`（padding 挤占）

---

## 目标效果

将设置面板从 **push-drawer（占据 flex 布局空间，挤压对话区）** 重构为 **non-modal 浮层 + 对话区空白挤占**：

- 视觉上是浮卡：左侧圆角、向左投影、悬浮于内容之上
- 行为上是非模态：**无遮罩**，对话区始终可交互（滚动消息、点击按钮、聚焦输入框等不被打断）
- 打开时，对话区内容容器的 `padding-right` 同步过渡到 `20rem`，让对话内容"被推开"
- 关闭方式仅限三种显式入口：齿轮按钮再次点击、面板 X 按钮、Escape 键。**不在外部点击时关闭**
- 动画使用 GPU 加速的 `transform: translateX`，替代低效的 `transition-[width]`
- 所有原有设置内容、状态、子面板逻辑保持不变

### 模态 vs 非模态的设计决策

模态（透明遮罩 + 点击外部关闭）会在面板打开时阻断对话区交互，体验类似一次性弹窗。本项目目标贴近 Claude desktop 的右侧伴随面板形态——用户可以一边翻阅消息、一边查看/调整设置，因此选择**非模态**：

- 不渲染遮罩层，省掉 backdrop DOM 节点
- 移除"点击外部关闭"行为；非模态面板被无意点击关掉是反直觉的
- 通过对话区 `padding-right` 动画让用户清晰感知面板挤占了空间

## 根因分析

### 当前结构

`HomePageLayout.tsx` 中 `<SettingsDrawer>` 是 flex 末尾子节点：

```tsx
<div className="flex h-screen w-screen overflow-hidden ...">
  <ConversationSidebar ... />
  <div className="flex min-w-0 flex-1 flex-col">   {/* 对话区 */}
    ...
  </div>
  <SettingsDrawer ... />   {/* w-0/w-80 占据 flex 布局空间 */}
</div>
```

### 当前动画（含 v1 已实施的两层 div 残留）

`SettingsDrawer.tsx` 最外层：

```tsx
<div className={clsx(
  "h-screen shrink-0 overflow-hidden",
  "transition-[width] duration-300 ease-in-out",
  isOpen ? "w-80" : "w-0"
)}>
  <div className="flex h-full w-80 flex-col rounded-l-2xl bg-surface shadow-[...]">
    {/* 抽屉内容 */}
  </div>
</div>
```

**问题**：

1. `transition-[width]` 是 layout property，触发布局重排，性能差于 GPU 加速的 transform
2. v1 双层 div 在 push-drawer 模式下勉强可用，但仍依赖宽度过渡，且无法与对话区做协调动画
3. 浮卡感受限——作为 flex 兄弟节点紧贴对话区，没有真正的"悬浮于内容之上"层次

---

## 推荐解决方案

### 总体策略

将 `SettingsDrawer` 改为单层 `position: fixed` 浮层，**脱离 flex 布局流**；同时给对话区内容容器加上 `padding-right` 过渡。两段动画相同时长、相同缓动、同步触发，视觉上是"浮层把对话区挤过去了"。

```
[fixed 浮层]              ← w-80 right-0 top-3 bottom-3，translate-x 动画
[对话区 content wrapper]  ← padding-right: 0 ↔ 20rem 动画
```

### 不再有遮罩

非模态形态下不存在 backdrop 节点。三种关闭入口：

1. 头部 X 按钮（已存在）
2. 齿轮按钮 toggle（已存在）
3. **Escape 键关闭**（新增）

### a11y 关键改进：`inert` 隔离关闭态

面板关闭时虽然 translate 移出屏幕，但 DOM 中所有可聚焦元素（input、button）仍可被 Tab 命中，屏幕阅读器仍可朗读。**必加** `inert`——React 19 原生支持，`@types/react` 19 类型声明已正确，可直接用 JSX 布尔属性。

## 具体改动

### 1. `frontend/src/app/home/components/SettingsDrawer.tsx`

#### 1.1 替换最外层 `return` 根节点

**改前**（v1 已实施版本，需完全替换）：

```tsx
return (
  <div className={clsx(
    "h-screen shrink-0 overflow-hidden",
    "transition-[width] duration-300 ease-in-out",
    isOpen ? "w-80" : "w-0"
  )}>
    <div className="flex h-full w-80 flex-col rounded-l-2xl bg-surface shadow-[-12px_0_32px_rgba(15,23,42,0.08)] dark:shadow-[-12px_0_32px_rgba(0,0,0,0.24)]">
      {/* 抽屉头部 */}
      {/* 抽屉内容 */}
    </div>
  </div>
)
```

**改后**：

```tsx
return (
  <div
    className={clsx(
      "fixed right-0 top-3 bottom-3 z-50",
      "flex w-80 flex-col overflow-hidden",
      "rounded-l-2xl",
      "bg-surface",
      "shadow-[-16px_0_40px_rgba(15,23,42,0.10),_-4px_0_12px_rgba(15,23,42,0.06)]",
      "dark:shadow-[-16px_0_40px_rgba(0,0,0,0.32),_-4px_0_12px_rgba(0,0,0,0.20)]",
      "transition-transform duration-300 ease-in-out",
      isOpen ? "translate-x-0" : "translate-x-full",
    )}
    aria-hidden={!isOpen}
    inert={!isOpen}
  >
    {/* 抽屉头部 —— 完全不变 */}
    {/* 抽屉内容 —— 完全不变 */}
  </div>
)
```

**关键参数**：

| 属性 | 值 | 说明 |
|------|-----|------|
| 定位 | `fixed right-0 top-3 bottom-3 z-50` | 贴右屏，上下各留 12px 形成悬浮感 |
| 宽度 | `w-80` | 320px，与对话区 padding-right 完全对齐 |
| 圆角 | `rounded-l-2xl` | 仅左侧两角圆角（右侧贴屏不需要） |
| 背景 | `bg-surface` | `#ffffff`，比页面底色 `bg-surface-alt` (`#F8F8F6`) 略亮 |
| 投影 | `-16px_0_40px / -4px_0_12px` 叠加 | 大投影 + 小投影模拟浮卡立体感；暗色加深 |
| 动画 | `transition-transform duration-300 ease-in-out` + `translate-x-0 ↔ translate-x-full` | GPU 加速 transform |
| a11y | `aria-hidden={!isOpen}` + `inert={!isOpen}` | 关闭时 DOM 中存在但屏幕阅读器/Tab 焦点不可达 |

#### 1.2 新增 Escape 键关闭逻辑

在组件主体（`handleDrawerClose` 定义之后）新增 `useEffect`：

```tsx
useEffect(() => {
  if (!isOpen) return
  const handleKeyDown = (e: KeyboardEvent) => {
    if (e.key === 'Escape') handleDrawerClose()
  }
  document.addEventListener('keydown', handleKeyDown)
  return () => document.removeEventListener('keydown', handleKeyDown)
}, [isOpen, handleDrawerClose])
```

> `handleDrawerClose` 已在原代码中闭包了 `persistContextFile` / `resetDrawerPanels` / `onClose` 收尾逻辑，可直接复用。该函数若不是稳定引用（每次 render 重建），effect 会重新订阅，但 `isOpen` 切换时本来就该重新订阅，可接受；不要为此引入 `useCallback`。

### 2. `frontend/src/app/home/components/HomePageLayout.tsx`

#### 2.1 给对话区内容容器加 padding-right 动画

**改前**：

```tsx
<div className="flex min-w-0 flex-1 flex-col">
  {/* 对话区内容 */}
</div>
```

**改后**：

```tsx
<div
  className={clsx(
    "flex min-w-0 flex-1 flex-col",
    "transition-[padding-right] duration-300 ease-in-out",
    isSettingsOpen ? "pr-80" : "pr-0",
  )}
>
  {/* 对话区内容 */}
</div>
```

**说明**：

- `pr-80` (`padding-right: 20rem`) 与 `SettingsDrawer` 的 `w-80` 完全一致
- duration 与 ease 与 `SettingsDrawer` 的 transform 动画一致，确保两端同步
- 若该 wrapper 已有其他 className，仅追加上述 3 行内容，原有不动

#### 2.2 SettingsDrawer 节点位置

`<SettingsDrawer>` 仍作为 flex 末尾子节点保留，**不移动**。它内部已是 `position: fixed`，自动脱离 flex 布局流，不占任何宽度——保留位置纯粹为减少 props 传递改动面。

### 3. 确认删除的代码

实施时确认以下 v1 残留全部消失，**不要任何形式残留**：

- 外层 `transition-[width]` 与 `w-0/w-80` 切换
- v1 添加的内层卡片 div（双层结构合并为单层 fixed）
- 与上一版 v2 草案不同：**不要**任何 backdrop 遮罩 JSX

## 验收标准

1. 点击齿轮图标，面板从右侧滑入；同时对话区内容容器 `padding-right` 从 0 平滑过渡到 `20rem`，对话内容向左压缩
2. 面板关闭时，反向同步动画：面板 translate 滑出右侧 + 对话区 padding 收回到 0
3. 面板打开期间，**对话区可正常交互**：滚动消息、点击消息按钮、聚焦输入框、发送消息等全部不被任何遮罩拦截
4. 面板背景色（`bg-surface` 白色）与页面底色（`bg-surface-alt` 暖灰）有可见层次区分；左侧圆角和向左投影清晰可见
5. **关闭方式仅限**：齿轮按钮再次点击、面板内 X 按钮、Escape 键。点击对话区任意位置不会关闭面板
6. 面板关闭时（`isOpen=false`），其内部 input、button **无法被 Tab 聚焦**，屏幕阅读器不朗读其内容（`inert` 生效）
7. 暗色模式下投影正常显示，不过度强烈
8. 内部子面板（Model selection、Assistant context）的 `absolute inset-0` 覆盖层正常工作
9. 面板 transform 过渡和对话区 padding 过渡保持完全同步（同一时长、同一缓动），无视觉错位
10. `tsc --noEmit` 通过；运行时控制台无 React warning（`inert` 在 React 19 + @types/react 19 下应直接接受布尔值，无需任何 ts-expect-error 注释）
