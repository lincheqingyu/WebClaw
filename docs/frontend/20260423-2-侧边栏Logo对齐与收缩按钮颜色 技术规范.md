# 侧边栏 Logo 对齐与收缩按钮颜色 技术规范

## 背景

截图反映 `ConversationSidebar` 顶部区域存在三处视觉问题：

1. **Logo 图标与右侧"Lecquy"文字未垂直居中对齐**：当前代码中存在两处像素级手动偏移（图片 `translate-y-[3px]`、文字 `pb-[3px]`），两个偏移方向相同且叠加，导致 logo 在视觉上低于网格 `items-center` 基准线，与右侧文字失去对齐。
2. **Logo 图标与下方"+"图标、"会话"图标未水平中心对齐**：三者都处于宽度为 `2.25rem`（36px）的左列，但"+"图标在展开态的容器尺寸为 `h-7 w-7`（28px），比列宽窄 8px，且缺少 `justify-self-center`，导致其在列内靠左对齐，水平中心偏左约 4px。
3. **收缩按钮颜色为灰色**：`PanelLeftClose` 按钮使用 `text-text-secondary`（次要文字色），在亮色模式下偏灰、不够清晰，应使用 `text-text-primary`（亮色模式黑色 / 暗色模式白色）。

## 涉及文件

- `frontend/src/app/home/components/ConversationSidebar.tsx`

## 问题定位

### 问题一：Logo 垂直对齐

**当前代码**（第 136–149 行）：

```tsx
<div className="inline-flex h-9 w-9 items-center justify-center justify-self-center overflow-visible">
  <img
    src={isDark ? '/logo-dark-mode.png' : '/logo-light-mode.png'}
    alt="Lecquy"
    className="block h-7.5 w-7.5 shrink-0 translate-y-[3px] scale-[1.24] object-contain"
    loading="eager"
  />
</div>

<div className="flex min-w-0 items-center">
  <div className="overflow-visible whitespace-nowrap pb-[3px] text-[1.5rem] font-semibold leading-[1.08] tracking-[-0.045em] text-text-primary">
    Lecquy
  </div>
</div>
```

**根因**：

- `translate-y-[3px]`：将 logo 图片向下平移 3px，脱离容器垂直中心
- `pb-[3px]`：给文字容器添加底部内边距，同方向偏移文字基线

两处偏移叠加，导致 logo 与文字视觉上均偏低，且两者之间也未对齐。

### 问题二："+" 图标水平中心不对齐

**当前代码**（第 181–188 行）：

```tsx
<span
  className={[
    'inline-flex shrink-0 items-center justify-center rounded-full text-text-primary',
    collapsed ? 'h-9 w-9' : 'h-7 w-7 bg-sidebar-active',
  ].join(' ')}
>
  <Plus className="size-5 shrink-0 translate-y-px" />
</span>
```

**根因**：

左列宽度为 `2.25rem`（36px），三个图标区域均在此列内：

| 元素 | 容器尺寸 | `justify-self` | 列内水平中心 |
|---|---|---|---|
| Logo 图标 | `h-9 w-9`（36px） | `justify-self-center` | 列中央 ✅ |
| "会话"图标 | `h-9 w-9`（36px，`collapsedIconShellClass`） | 默认 stretch，36px 恰好填满列 | 列中央 ✅ |
| "+"图标（展开态） | `h-7 w-7`（28px） | 无 `justify-self-center`，靠左对齐 | 偏左 4px ❌ |

"+"的 `span` 比列宽窄 8px，且没有指定 `justify-self-center`，在 CSS Grid 中默认靠起始边排列，导致圆圈中心偏左。

### 问题三：收缩按钮颜色

**当前代码**（第 151–162 行）：

```tsx
<button
  ...
  className={[
    'inline-flex h-9 w-9 shrink-0 items-center justify-center justify-self-end rounded-xl text-text-secondary',
    'transition-colors hover:bg-sidebar-hover hover:text-text-primary',
  ].join(' ')}
>
  <PanelLeftClose className="size-[18px] shrink-0" />
</button>
```

**根因**：`text-text-secondary` 是灰色次要文字色，用于图标会导致按钮在视觉上不够清晰，与操作按钮的语义不符。hover 态才切换到 `text-text-primary` 的逻辑也让默认状态过于"隐身"。

## 修改方案

### 修改一：移除 Logo 与文字的手动偏移

**修改点 A — logo 图片**：移除 `translate-y-[3px]`，保留其余类。

```tsx
// 修改前
className="block h-7.5 w-7.5 shrink-0 translate-y-[3px] scale-[1.24] object-contain"

// 修改后
className="block h-7.5 w-7.5 shrink-0 scale-[1.24] object-contain"
```

**修改点 B — Lecquy 文字容器**：移除 `pb-[3px]`，保留其余类。

```tsx
// 修改前
className="overflow-visible whitespace-nowrap pb-[3px] text-[1.5rem] font-semibold leading-[1.08] tracking-[-0.045em] text-text-primary"

// 修改后
className="overflow-visible whitespace-nowrap text-[1.5rem] font-semibold leading-[1.08] tracking-[-0.045em] text-text-primary"
```

> **执行建议**：移除后先肉眼验证 logo 与文字是否与 grid `items-center` 自然对齐。若 logo 素材本身存在内嵌白边或不对称内边距，可酌情调整图片容器的 `h-` / `w-` 或使用 `object-position`，**不应再引入 `translate-y` 或 `pb` 的方向性偏移**。

### 修改二："+" 图标添加 `justify-self-center`

**修改点 — "新建会话"按钮内的图标 span**（第 181–185 行）：在展开态（非 `collapsed`）时添加 `justify-self-center`，使 28px 的圆圈在 36px 的列内水平居中。

```tsx
// 修改前
collapsed ? 'h-9 w-9' : 'h-7 w-7 bg-sidebar-active'

// 修改后
collapsed ? 'h-9 w-9' : 'h-7 w-7 justify-self-center bg-sidebar-active'
```

> **说明**：折叠态（`collapsed`）下"+"图标已是 `h-9 w-9`，与列宽完全匹配，无需改动。

### 修改三：收缩按钮改为 `text-text-primary`

```tsx
// 修改前
className={[
  'inline-flex h-9 w-9 shrink-0 items-center justify-center justify-self-end rounded-xl text-text-secondary',
  'transition-colors hover:bg-sidebar-hover hover:text-text-primary',
].join(' ')}

// 修改后
className={[
  'inline-flex h-9 w-9 shrink-0 items-center justify-center justify-self-end rounded-xl text-text-primary',
  'transition-colors hover:bg-sidebar-hover',
].join(' ')}
```

说明：

- 默认色从 `text-text-secondary`（灰）改为 `text-text-primary`（亮色模式黑 / 暗色模式白）
- hover 态的 `hover:text-text-primary` 已无必要，移除即可，`hover:bg-sidebar-hover` 保留提供背景反馈

## 验证点

- [ ] 亮色模式：logo 图标与"Lecquy"文字垂直居中，无明显偏高或偏低
- [ ] 暗色模式：logo 图标与"Lecquy"文字垂直居中，无明显偏高或偏低
- [ ] 展开态：logo 图标、"+"图标、"会话"图标三者左列水平中心对齐（肉眼垂直线一致）
- [ ] 亮色模式：收缩按钮默认为黑色，hover 时出现背景色
- [ ] 暗色模式：收缩按钮默认为白色，hover 时出现背景色
- [ ] 折叠态：各图标对齐不受影响（折叠态均为 `h-9 w-9`，无需额外处理）
- [ ] 侧边栏折叠 / 展开动画过程中，顶部区域无布局跳动

## 不纳入本次

- logo 图片素材本身的内边距问题（如有，需单独处理图片资源）
- 折叠态（`collapsed=true`）的展开按钮 `PanelLeftOpen`——其颜色当前已为 `text-text-primary`，无需修改
