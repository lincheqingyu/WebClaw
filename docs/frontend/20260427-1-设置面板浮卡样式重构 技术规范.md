# 设置面板浮卡样式重构

> 本文档为交付给 Claude Opus 的改造指令。要求**完整实现目标效果**，不打补丁，不堆叠临时修复。

---

## 目标效果

设置面板打开时，对话区向左收缩（保留现有 flex 布局占位行为），但面板本身呈现为一张**从右侧滑入的浮卡**：

- 左边缘带圆角（只有左侧两个角有圆角，右边缘贴屏幕边）
- 左边缘有向左的投影，制造"悬浮"的立体感
- 面板与对话区之间**没有竖线分隔符**
- 面板背景与页面底色有轻微的视觉层次区分

---

## 根因分析

**文件**：`frontend/src/app/home/components/SettingsDrawer.tsx`（第 616–622 行）

当前最外层容器：

```tsx
<div
  className={clsx(
    "h-screen shrink-0 overflow-hidden bg-surface-alt",
    "transition-[width] duration-300 ease-in-out",
    isOpen ? "w-80" : "w-0"
  )}
>
```

问题在于：**布局占位容器直接承担了视觉样式**（`bg-surface-alt`）。当 `w-0` 时背景被裁切，但没有任何形态上的"卡片感"。面板与对话区之间因 flex 并排而产生硬边，没有投影或圆角。

---

## 推荐解决方案

将**布局占位层**与**视觉卡片层**分离，用两层 div 实现：

```
[外层 div]  ← 布局占位，w-0/w-80 过渡，overflow-hidden，透明，无背景
  [内层 div] ← 视觉卡片，固定 w-80，圆角左边，左侧投影，背景色
    [原有内容不变]
```

**外层**：只负责宽度动画和裁切，背景透明
**内层**：固定 `w-80`（不参与过渡），承载所有视觉样式

这样做的优势：
- 内层卡片始终保持完整宽度和圆角形态，outer `overflow-hidden` 负责裁切入场动画
- 不需要改动任何内部内容的结构和逻辑
- 无需在 `HomePageLayout.tsx` 里做任何修改
- 改动范围最小，只涉及 `SettingsDrawer.tsx` 最外层两行

---

## 具体改动要求

**文件**：`frontend/src/components/ui/SettingsDrawer.tsx`（注意：实际路径为 `frontend/src/app/home/components/SettingsDrawer.tsx`）

### 修改最外层 `return` 的根节点

**改前**：
```tsx
<div
  className={clsx(
    "h-screen shrink-0 overflow-hidden bg-surface-alt",
    "transition-[width] duration-300 ease-in-out",
    isOpen ? "w-80" : "w-0"
  )}
>
  {/* 抽屉头部 */}
  ...
  {/* 抽屉内容 */}
  ...
</div>
```

**改后**：
```tsx
<div
  className={clsx(
    "h-screen shrink-0 overflow-hidden",
    "transition-[width] duration-300 ease-in-out",
    isOpen ? "w-80" : "w-0"
  )}
>
  <div className="flex h-full w-80 flex-col rounded-l-2xl bg-surface shadow-[-12px_0_32px_rgba(15,23,42,0.08)] dark:shadow-[-12px_0_32px_rgba(0,0,0,0.24)]">
    {/* 原有内容完全不变，整体平移到这里 */}
    ...
  </div>
</div>
```

### 关键参数说明

| 属性 | 值 | 说明 |
|------|-----|------|
| 外层背景 | 无（透明） | 不再有 `bg-surface-alt`，外层只做裁切 |
| 内层宽度 | `w-80`（固定） | 不随外层动画变化，始终是完整卡片 |
| 内层圆角 | `rounded-l-2xl` | 仅左侧两个角有圆角（`border-radius` 只在 left 方向） |
| 内层背景 | `bg-surface` | 比页面底色 `bg-surface-alt` 略亮一档，形成层次感 |
| 内层投影 | `shadow-[-12px_0_32px_rgba(15,23,42,0.08)]` | 向左投影，模拟浮卡效果；暗色模式加深透明度 |

> `bg-surface` 在浅色模式为 `#ffffff`，`bg-surface-alt` 为 `#F8F8F6`，层次差异足够但不强烈，符合设计目标。

## 不需要改动的内容

- `HomePageLayout.tsx`：flex 布局结构、`<SettingsDrawer>` 的调用方式均不变
- `SettingsDrawer.tsx` 内部所有逻辑、状态、子面板（Model panel、Context panel）完全不变
- `index.css`：无需新增任何样式
- 动画机制（`transition-[width]`）保持现有实现，不替换

---

## 验收标准

1. 设置面板打开时，对话区向左收缩，二者之间无竖线
2. 面板左边缘可见圆角和向左的投影阴影
3. 面板背景色（`bg-surface`）与页面底色（`bg-surface-alt`）有轻微区分
4. 关闭动画（宽度收缩至 0）正常，内容被平滑裁切
5. 暗色模式下投影可见，不过度强烈
6. 内部子面板（Model selection、Assistant context）的绝对定位覆盖层（`absolute inset-0`）仍然正常工作，不超出面板边界
