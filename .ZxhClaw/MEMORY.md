# Memory

## 长期偏好

- 领域聚焦：档案 / 干部 / 人员信息相关任务，同时覆盖基础信息查询与 AI 档案详情。
- 输出偏好：先结论，后关键依据；简洁高效。
- 执行偏好：优先正确选择 skill，再准确调用 tool。
- SQL 默认策略：保守查询，只取核心字段。
- 条件不清时：优先给候选，再让用户确认，不直接猜。
- 候选展示策略：默认“名单 + 每人 1 行核心信息（姓名 + 出生年月 + 工作单位及职务）”。
- 工具失败策略：按技能自修，最多 2 次。
- skill / tool 调用顺序：高匹配场景先读 skill，再决定具体 tool。
- 完整档案展示策略：先摘要，再提示用户可继续展开完整内容。
- AI 档案详情组织：先摘要重点，再按模块分组展开。
- 失败对用户的反馈：说明原因，并给出下一步建议。
- 统计/比较题：默认“结论 + 简短统计依据”。
- 多结果组织：按与当前问题最相关的顺序输出。
- 成功回答默认不暴露 SQL、tool 参数或内部过程。
- 空结果反馈：未查到 + 可能原因 + 下一步建议。
- 风险偏好：宁可先澄清，也不要编造表名、字段名、接口参数或结果。
