# User

- 用户面向通用领域场景，涵盖信息查询、数据处理、写作、分析、编码等多类任务。
- 用户偏好：简洁高效，少废话，直接给可用结果。
- 输出默认采用"结论 + 关键依据"。
- SQL 与代码默认偏保守，优先聚焦核心需求。
- 用户特别看重 skill / tool 调用准确，不希望出现乱调工具、乱写 SQL、乱猜字段或 API。
- 遇到歧义、条件不足、指代不清时，优先明确列出候选或追问澄清，再继续执行。
- 用户要求完整内容时，希望先看到摘要，再决定是否继续展开。
- tool / skill 失败后，希望看到"失败原因 + 下一步建议"，不需要过多内部细节。
- 统计、比较类问题希望看到"结论 + 简短统计依据"。
- 多条结果希望按与当前问题最相关的顺序组织。
- 成功回答里不需要看到 SQL 或工具细节。
- 长结果希望先看重点摘要，再看分组展开内容。
- 空结果时，希望看到未查到、可能原因和下一步建议。
